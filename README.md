### Описание проекта

Веб‑сайт на основе `Html` и `Flask`, созданный для поиска игры и приятного времяпровождения
Функционал:

- **Ответы на вопросы**: Пользователи могут задать вопрос по игре, на который дадут ответ другие люди
- **Поиск команды**: Пользователи ищут команду для комфортной игры
- **Общение**: Пользователи могут делиться информацией/вПпечатлениями о игре с другими людьми или просто общаться с ними
- **Пользователи**: регистрация, авторизация, выход из системы.
- **Роли**:
  - **Администратор**: полный доступ к удалению сообщений, вопросов, некультурных комментариев, выдаче мута/бана пользователю (не дает полностью использовать сайт)
  - **Обычный пользователь**: регистрация, вход, общение, возможность найти команду/ответ на вопрос


Данные хранятся в базе **SQLite** (`main.db`) через ORM **SQLAlchemy**.

---

### Используемые технологии

- **Язык**: Python 3 (проект проверен на Python 3.13).
- **Веб‑фреймворк**: `Flask`
  - роутинг, шаблоны (`Jinja2`), флеш‑сообщения.
- **Аутентификация**: `flask_login`
  - сессии, объект `current_user`, декоратор `user_loader`.
- **ORM и БД**: `flask_sqlalchemy` + `SQLAlchemy`
  - база `sqlite:///main.db` (файл в папке `instance/`).
- **Безопасность паролей**: `werkzeug.security`
  - `generate_password_hash`, `check_password_hash`.
- **Шаблоны и статика**:
  - HTML‑шаблоны в `templates/`
  - статика (картинки и т.п.) в `static/`

Все зависимости перечислены в `requirements.txt`

### Возможности веб‑интерфейса

- **Главная страница (`/`)**
  - быстрое обучение по сайту
- **Общение (`/community`)**
  - только для **авторизованного человека**;
  - общение с пользователями;
  - возможность ставить реакции пользователям/писать коментарии
  - если пользователь — админ, отображаются действия для удаления.
- **Поиск команды (`/ban/<int:user_id>`)** 
  - только для админа;
  - возможность создать 'анкету - приглашение' в комнду в игре с выбором характеристик.
- **Бан (`/team`)** 
  - только для **авторизованного человека**;
  - возможность создать 'анкету - приглашение' в комнду в игре с выбором характеристик.
- **Поиск команды (`/team`)** 
  - только для **авторизованного человека**;
  - возможность создать 'анкету - приглашение' в комнду в игре с выбором характеристик.
- **Лайки (`//community/like/<int:message_id>`)**
  - для пользователя, не писавший данный комментарий 
- **Репутация (`/profile/<int:user_id>/reputation`)**
  - возможность 'похвалить другого пользователя за те или иные действия'
- **Профиль (`/profile/<int:user_id>`)**
  - возможность посмотреть свою статистику или статистику другого пользователя 
- **Удаление сообщений ('/community/delete/<int:message_id>')**
  - только для администратора или пользователя, отправящего данное сообщение.
- **Удаление новости (`/post/<id>/del`)**
  - только для администратора.
- **Регистрация (`/registration`)**
  - проверка уникальности логина;
  - проверка совпадения пароля и подтверждения;
  - пароли сохраняются в виде хэша.
- **Авторизация (`/login`)**
  - вход по логину и паролю;
  - проверка хэша пароля;
  - если уже авторизован — редирект на главную.
- **Выход (`/logout`)**

### Инструкция для запуска на windows (с виртуальным окружением)

#### 1. Клонирование / копирование проекта

Если проект уже есть на рабочем столе, этот шаг можно пропустить.  
Если используете GitHub:

```powershell
git clone https://github.com/RealAnski1/Repo
cd Repo
```

#### 2. Создание и активация виртуального окружения

```powershell

# создать виртуальное окружение (папка venv появится в корне проекта)
python -m venv venv

# активировать окружение
.\venv\Scripts\Activate.ps1
```

При успешной активации в начале строки PowerShell появится префикс `(venv)`.

#### 3. Установка зависимостей

```powershell
pip install --upgrade pip
pip install -r requirements.txt
```

#### 4. Запуск приложения

Запуск основного Flask‑приложения:

```powershell
python __init__.py
```

По умолчанию сервер стартует на:

- `http://127.0.0.1:5000/` — веб‑интерфейс
- `http://127.0.0.1:5000/api/...` — REST API

#### 5. Остановка приложения

В окне PowerShell нажмите `Ctrl + C`.

### REST API (краткое описание)

API подключено в `__init__.py` через функцию `init_api(app, db, Article, User)` из `api.py`.  
Базовый префикс: `/api`.

Основные эндпоинты:

- **GET `/api/profile`**
  - список пользователей;
  - ответ в JSON: `{ "ok": true, "items": [...] }`.
- **GET `/api/profile/<id>`**
  - получение одного человека;
  - `404`, если не найдено.
- **POST `/api/community`**
  - получение всех сообщений
- **POST `/api/community/<id>`** 
  - получение конкретного сообщения

Пример работы с API есть в скрипте `api_example.py`:

```bash
python api_example.py
```

Скрипт:

- получает список сообщений/профилей;
- запрашивает ID для просмотра;

### Работа с базой данных

- Файл базы: `instance/main.db`.
- При первом запуске (или если файла нет) таблицы создаются автоматически командой `db.create_all()`.
- Модели `Article` и `User` описаны в `__init__.py`.
- Для полного сброса данных можно удалить файл `instance/main.db` (при следующем запуске он будет создан заново с пустыми таблицами).
